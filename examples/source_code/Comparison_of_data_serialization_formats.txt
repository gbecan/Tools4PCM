*c. <span class="citation wikicite" id="endnote_thrifttxt"><b>[[#ref_thrifttxt|^]]</b></span>Theoretically possible due to abstraction, but no implementation is included.
*d. <span class="citation wikicite" id="endnote_pbtextformat"><b>[[#ref_pbtextformat|^]]</b></span>The primary format is binary, but a text format is available.
*e. <span class="citation wikicite" id="endnote_stdrefs"><b>[[#ref_stdrefs|^]]</b></span>Means that generic tools/libraries know how to encode, decode, and dereference a reference to another piece of data in the same document. A tool may require the [[Interface description language|IDL]] file, but no more. Excludes custom, non-standardized referencing techniques.
*f. <span class="citation wikicite" id="endnote_asn1refs"><b>[[#ref_asn1refs|^]]</b></span>ASN.1 does offer [[Object identifier|OIDs]], a standard format for globally unique identifiers, as well as  a standard notation ("absolute reference") for referencing a component of a value. Thus it would be possible to reference a component of an encoded value present in a document by combining an OID (assigned to the document) and an "absolute reference" to the component of the value. However, there is no standard way to indicate that a field contains such an absolute reference. Therefore, a generic ASN.1 tool/library cannot automatically encode/decode/resolve references within a document without help from custom-written program code.
