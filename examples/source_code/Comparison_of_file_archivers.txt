=== Operating system support ===
The [[operating system]]s the archivers can run on without emulation or compatibility layer. Linux Ubuntu's own GUI ''Archive manager,'' for example, can  open and create many archive formats (including Rar archives) even to the extent of splitting into parts and encryption and ability to be read by the ''native program''. This is presumably a "compatibility layer."

'''Notes:'''
# <span class="citation wikicite" id="endnote_decompress-only"><b>[[#ref_decompress-only|^]]</b></span> Application is only for decompression.
# <span class="citation wikicite" id="endnote_tar-compression"><b>[[#ref_tar-compression|^]]</b></span> [[GNU]] tar calls the external programs gzip and bzip2 to perform compression; these external programs usually come with systems that contain GNU tar.
# <span class="citation wikicite" id="endnote_features-WinRAR"><b>[[#ref_features-WinRAR|^]]</b></span> Allows adding a variable amount of [[redundancy (information theory)|redundancy]] for much better error recovery. See also [[RAR (file format)]].
# <span class="citation wikicite" id="endnote_unicode-names"><b>[[#ref_unicode-names|^]]</b></span> Extracting/adding file and/or directory names into archive in either [[UTF-7]], [[UTF-8]] or [[UTF-16/UCS-2]] encoding to support single file/directory name which contains characters from different languages.  More recent versions of the zip file format have support for Unicode filenames.
# <span class="citation wikicite" id="endnote_winrar-7z"><b>[[#ref_winrar-7z|^]]</b></span> In WinRAR 3.60, when opening 7-Zip archives which contains Unicode file/directory names, they will not be displayed correctly. There will be no problem extracting them, however.
# <span class="citation wikicite" id="endnote_powerarchiver"><b>[[#ref_powerarchiver|^]]</b></span> Does support Unicode names, but not under the default (initial) option settings: the user must tick "Use OEM conversion for filenames" under "General" on the "Miscellaneous" tab in the Configuration dialog to enable Unicode name support. Full support for Unicode files names by default is supported only for 7-Zip and RAR archive formats.
# <span class="citation wikicite" id="endnote_features-ALZip"><b>[[#ref_features-ALZip|^]]</b></span> Commandline batch compression is available only for ZIP and ALZ formats.
# <span class="citation wikicite" id="endnote_unicode-peazip"><b>[[#ref_unicode-peazip|^]]</b></span> UTF-8 support was completed in release 2.5. On Unix systems the support is full, while on Windows systems, due to limitation of the Windows shell environment, double-character encoded characters cannot currently be displayed in filenames of archived objects, and those chars are replaced by a [[Mojibake|corrupted]] character. Optionally, extended characters can be set to be always replaced by a 'corrupted' "?" character to avoid possible issues between archive's and system's character encodings.
# <span class="citation wikicite" id="endnote_batch-winzip"><b>[[#ref_batch-winzip|^]]</b></span> Commandline batch compression and expansion requires free add-on software downloaded from the WinZip website.<sup class="reference plainlinks nourlexpansion" id="ref_{{{1}}}">[//en.wikipedia.org/wiki/Special:ExpandTemplates#endnote_]</sup>
# <span class="citation wikicite" id="endnote_peazip-encryption"><b>[[#ref_peazip-encryption|^]]</b></span> Peazip supports file encryption and file name encryption, although only in certain types of archives, including its own Pea format, 7-zip, zip and Arc.
# <span class="citation wikicite" id="endnote_infozip-shell"><b>[[#ref_infozip-shell|^]]</b></span> Many shells have built-in zip file support. [[Windows Explorer]] has "Send To"->"ZIP-compressed folder".
# <span class="citation wikicite" id="endnote_infozip-since30"><b>[[#ref_infozip-since30|^]]</b></span> Since version 3.0 (2008).

'''Notes:'''
# <span class="citation wikicite" id="endnote_reading-7zip"><b>[[#ref_reading-7zip|^]]</b></span> Used to, but no longer does, due to technical and legal issues.
# <span class="citation wikicite" id="endnote_tar-compression"><b>[[#ref_tar-compression|^]]</b></span> Tar implementations call external programs (like [[compress]], [[gzip]] or [[bzip2]] or any other programs working with [[Stream (computing)|abstract streams]] and supporting the "-d" option) to perform (de)compression, and allowing you to implement your own filters<sup class="reference plainlinks nourlexpansion" id="ref_tar-custom">[//en.wikipedia.org/wiki/Special:ExpandTemplates#endnote_tar-custom]</sup>. These external programs may be shipped with your Operating System.
# <span class="citation wikicite" id="endnote_tar-custom"><b>[[#ref_tar-custom|^]]</b></span> GNU tar lets you implement your own filters [http://www.gnu.org/software/tar/manual/tar.html#index-use_002dcompress_002dprogram-460], allowing you to use other compression programs (p7zip, ...) and filters (GPG, ...).
# <span class="citation wikicite" id="endnote_reading-winzip"><b>[[#ref_reading-winzip|^]]</b></span> Starting from version 11.
# <span class="citation wikicite" id="endnote_reading-winzip"><b>[[#ref_reading-winzip|^]]</b></span> Requires external program. [http://www.winzip.com/wzdifs.htm More info]
# <span class="citation wikicite" id="endnote_ArchiveUtility-ISO"><b>[[#ref_ArchiveUtility-ISO|^]]</b></span> Archive Utility itself is unable to open ISO files, but [[Disk Utility]], which also comes with Mac OS X, is able to mount them as virtual disks.
# <span class="citation wikicite" id="endnote_ZipGenius-external"><b>[[#ref_ZipGenius-external|^]]</b></span> Requires external program.[http://www.zipgenius.it/eng/?page_id=3]
# <span class="citation wikicite" id="endnote_Unarchiver-SITX"><b>[[#ref_Unarchiver-SITX|^]]</b></span> Only partial support for reading proprietary SITX format. [http://wakaba.c3.cx/sup/kareha.pl/1151796773/]
# <span class="citation wikicite" id="endnote_reading-arc"><b>[[#ref_reading-arc|^]]</b></span> FreeARC uses <tt>.arc</tt> as its filename extension, but this format is not the same as the [[ARC (file format)|traditional ARC file]].
# <span class="citation wikicite" id="endnote_stream-compression"><b>[[#ref_stream-compression|^]]</b></span> Supports these formats as compression stream of other archive formats like tar.bz2 or iso.xz but does not support the format as an archive itself.
# <span class="citation wikicite" id="endnote_B1 Free Archiver-ARJ"><b>[[#ref_B1 Free Archiver-ARJ|^]]</b></span>B1 Free Archiver for Android and B1 Online Archiver support ARJ format.

==== Uncommon archive format support ====
PeaZip has full support for various LPAQ and [[PAQ]] formats, QUAD and BALZ (highly efficient [[ROLZ]] based compressors), [[FreeArc]] format, and for its native [[List of archive formats|PEA]] format.
 
7-Zip includes read support for [[.msi]], [[cpio]] and [[xar (archiver)|xar]], plus Apple's [[Apple Disk Image|dmg]]/[[Hierarchical File System|HFS]] disk images and the [[deb (file format)|deb]]/[[RPM Package Manager|.rpm]] package distribution formats; beta versions (9.07 onwards) have full support for the LZMA2-compressed .[[xz]] format.
